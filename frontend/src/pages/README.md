# pages

<!--
  This folder contains route-level screens.

## Purpose

Host top-level screen components mapped to routes.

## Responsibilities

- Compose feature modules into screens
- Keep route-specific logic out of shared components
- Serve as the entry point for page-level navigation

## Example usage

Create pages for upload, dashboard, AI chat, and requirement detail views.
