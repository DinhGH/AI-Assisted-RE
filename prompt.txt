# 🧠 SYSTEM CONTEXT

This system analyzes software requirements using AI:

* Requirement upload (file/manual)
* NLP analysis (actor/action/object)
* Ambiguity detection
* Consistency & contradiction detection
* Quality scoring
* AI chat for refinement
* Dashboard visualization

---

# 🎯 CORE UI REQUIREMENT (MANDATORY)

You MUST implement a **NotebookLM-style 3-column workspace**:

### LEFT — Requirements Panel

* List of requirements
* Select / Add / Delete / Upload
* Drives entire system state

### CENTER — Main Workspace

* Initial: Upload / Manual input
* After: Chat interface
* Context-aware (based on selected requirement)

### RIGHT — Analysis Panel

* Top: Score visualization
* Bottom: Editable requirement + actions

### DASHBOARD (STATE-BASED)

* NO routing
* Toggle via state
* Show charts + aggregated data
* Export PDF

# 🎨 UI DESIGN RULES

## Typography

* Use distinctive fonts (NO Inter, Arial, Roboto)
* Combine display + readable font

## Color System

* Use CSS variables
* Strong contrast
* Avoid generic gradients

## Layout

* Asymmetry is encouraged
* Clear hierarchy
* Independent scroll per column

## Motion

* Subtle but meaningful
* Hover, transitions, panel changes
* Avoid over-animation

## Visual Identity

* Must feel like a real product
* Not generic dashboard

# ⚙️ DATA FLOW (VERY IMPORTANT)

  → triggers backend call
  → updates:

  * chat (center)
  * score (right top)
  * editor (right bottom)

* Editing requirement:
  → update backend
  → update UI immediately

* Switching dashboard:
  → NO data loss
